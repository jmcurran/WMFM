# Model explanation tab and tutor rebuild

## Context

I am working on an R package/app called WMFM.

There is strong evidence that an earlier branch called `explaining-explanations` contained unmerged work related to a dedicated model explanation surface in the app, including a guided tutor helper. That branch should not now be merged directly, because the current code base has diverged substantially and a direct merge would likely create conflicts and reintroduce outdated structure.

This context is for planning and implementing a rebuild of the lost model explanation tab and tutor functionality against the current architecture.

---

## Goal

Rebuild a dedicated student-facing model explanation tab and guided tutor layer in the current WMFM app.

This rebuilt feature should:
- give students a clear place in the app to understand how their explanation was produced
- present explanation-related information in a structured, educational way
- incorporate current explanation outputs and any rebuilt audit/teaching-summary layers
- avoid exposing raw developer-facing internal objects directly
- fit the current app layout and server logic cleanly

This is a rebuild task, not a merge task.

---

Based on the old branch history, the earlier branch likely included:
- a dedicated Model Explanation tab

- integration between explanation content and UI surfaces designed to help users interpret model explanations

The commit history strongly suggests that this was an intentional user-facing feature rather than a small internal helper.

Likely old-branch files of interest include:

---

## Constraints and requirements

- Do not merge the historical branch wholesale.
- Rebuild against the current app architecture.
- Do not force old UI patterns back into the app if the current layout has moved on.
- Keep the feature student-facing and educational.
- Use plain language and clear structure.
- Preserve current app behaviour where possible.
- Design the tab so it can gracefully consume:
  - explanation text
  - any rebuilt audit trail
  - any rebuilt teaching summary
- Follow project style:
  - use `=` for assignment
  - use camelCase
  - always use braces
  - use roxygen2 for documentation where relevant
- Add app-focused tests where practical.

## Suggested interpretation of the feature

The rebuilt Model Explanation tab should likely be the place where students can see:
- the main generated explanation
- a teaching summary that helps them read it
- a how-this-was-constructed view based on current structured data
- guided prompts or tutor-style framing that help them understand what the app is doing

The guided tutor should not feel like a developer debugger. It should feel like a teaching aid.

Possible responsibilities of the tutor layer:
- explain what kind of quantity is being interpreted
- point out the reference point or baseline used
- explain what change in a predictor means in the explanation
- clarify whether interpretation is additive, multiplicative, probability-based, odds-based, or on another scale
- help connect the explanation back to the research question

## Rebuild strategy

### Stage 1: inspect old UI and identify the intended user experience

Tasks:
- inspect old versions of:
  - `R/app-ui.R`
  - `R/app-server.R`
  - `R/app-explanationTutor.R`
  - explanation-related UI helper files
- determine:
  - what the old Model Explanation tab contained
  - how the tutor helper was invoked
  - what reactive inputs and outputs it depended on
  - which parts still make sense for the current app

Deliverable:
- a concise note describing the intended user experience and major UI components to recover

### Stage 2: define the rebuilt tab responsibilities

Tasks:
- decide what belongs in the dedicated Model Explanation tab in the current app
- separate responsibilities clearly between:
  - raw explanation display
  - teaching summary
  - explanation construction guidance
  - tutor-style prompts or notes
- define which parts are mandatory and which are optional depending on data availability

Deliverable:
- a simple functional specification for the tab layout and tutor behaviour

### Stage 3: define current reactive/data inputs

Tasks:
- inspect current server logic to identify the authoritative sources of:
  - fitted model outputs
  - explanation text
  - audit information
  - teaching-summary information if rebuilt separately
- define a clean data flow into the tab
- avoid duplicating reactive computations unnecessarily

Deliverable:
- a current-architecture data flow design for the Model Explanation tab

### Stage 4: rebuild tutor helper logic

Tasks:
- implement or rewrite `R/app-explanationTutor.R` or equivalent helpers
- keep tutor behaviour simple, deterministic where possible, and modular
- ensure it can consume current explanation-related data without depending on obsolete infrastructure
- separate tutor message generation from rendering where practical

Deliverable:
- tutor helper logic compatible with the current app

### Stage 5: rebuild UI layout

Tasks:
- implement the Model Explanation tab in the current `app-ui.R`
- ensure the layout is clear, readable, and educational
- determine how to present:
  - explanation text
  - teaching summary
  - audit/how-constructed information
  - tutor guidance
- keep visual structure straightforward and robust

Deliverable:
- a rebuilt dedicated explanation tab in the app UI

### Stage 6: integrate server-side rendering

Tasks:
- wire the rebuilt UI into `app-server.R`
- add rendering and fallback behaviour
- ensure the tab updates correctly when the model or explanation changes
- verify that missing pieces do not crash the UI

Deliverable:
- stable server integration for the explanation tab and tutor

### Stage 7: testing and polish

Tasks:
- add or update app-level tests around:
  - tab presence
  - output rendering
  - conditional display behaviour
  - tutor output availability
- add targeted helper tests if the tutor logic is implemented in standalone functions
- polish wording and ordering after behaviour is stable

Deliverable:
- passing tests and a stable student-facing explanation UI
